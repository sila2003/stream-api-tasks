PGO ENG 12 – Advanced Java Stream API Exercises

Introduction

This repository contains a complete implementation of the Stream API laboratory assignment prepared for the PGO ENG 12 course.

The purpose of this project is to demonstrate practical usage of modern Java functional programming techniques using the Java Stream API. The project was developed using Java 17 and focuses on writing efficient, readable, and maintainable stream-based code.

The assignment simulates a small order management system where collections of orders, products, customers, and statistics are processed using stream pipelines and collectors.


Technologies Used

• Java 17  
• Java Stream API  
• IntelliJ IDEA  
• Git & GitHub  

No external libraries or frameworks were used in this project.


Project Objectives

The primary goal of this assignment was to gain practical experience with:

• filtering collections using predicates  
• transforming data with map  
• flattening nested structures using flatMap  
• sorting stream results  
• grouping and partitioning data  
• collecting stream results into lists and maps  
• working with Optional and OptionalDouble  
• generating statistical summaries  
• writing clean functional-style Java code  


Project Description

The application represents a simplified order management system.

Each order contains:

• order ID  
• customer name  
• order date  
• list of ordered items  
• order status  

Each order item contains:

• product  
• quantity  
• total item price  

Products contain information about:

• product name  
• category  
• product price  

The system processes collections of orders and generates reports, summaries, statistics, and analytical results using Stream API operations.

Implemented Tasks

Task 1 – Active Order IDs  
Extract all order IDs from orders whose status is not CANCELLED.

Task 2 – Orders Above Selected Value  
Return orders whose total value exceeds a specified minimum amount and sort them by total value in descending order.

Task 3 – Unique Customers  
Generate an alphabetically sorted list of unique customer names.

Task 4 – Products From Active Orders  
Extract unique product names from all active orders.

Task 5 – Total Revenue  
Calculate the total revenue generated by all non-cancelled orders.

Task 6 – Average Delivered Order Value  
Compute the average value of delivered orders using OptionalDouble.

Task 7 – Count Orders By Status  
Group orders by status and count how many orders belong to each category.

Task 8 – Revenue By Product Category  
Calculate total revenue generated by each product category.

Task 9 – Top Customers  
Find customers who spent the most money and return the top results.

Task 10 – Partition Active Orders By Value  
Partition active orders into expensive and regular groups using a value threshold.

Task 11 – Most Expensive Delivered Order  
Find the delivered order with the highest total value.

Extra Task – Statistical Summary  
Generate complete statistical information for active orders using DoubleSummaryStatistics.
